Tomita - парсер для извлечения структурированных данных из текста на естественном языке. С его помощью мы:
а) выбираем из текста только самостоятельные части речи (существительное, глагол, прилагательное, наречие);
б) приводим слова к начальной форме.

В файле test.txt - текст на естественном языке для проверки работы парсера. Для начала работы в командную строку в директиве с томитой и файлами нужно ввести
В Linux, FreeBSD и прочих *nix системах:
./tomitaparser config.proto
В Windows:
tomitaparser.exe config.proto
где "tomitaparser" - соответствующая версия томиты https://tech.yandex.ru/tomita/ (его нужно скачать отдельно)
После этого в файлах facts.txt и PrettyOutput.html можно посмотреть результат.

Текущие проблемы:
а) не снимает неоднозначность (число "три" трактуется как глагол и приводится к форме "тереть")
б) хотя грамматика представлена в виде

S -> Word<gram='A'> interp (Word.Norm::norm="nom,sg");
S -> Word<gram='ADV'> interp (Word.Norm);
S -> Word<gram='S'> interp (Word.Norm::norm="nom,sg");
S -> Word<gram='V'> interp (Word.Norm::norm="inf");

Иногда вытаскивает предлоги, союзы и местоимения и не всегда переводит слово в нужную форму.
